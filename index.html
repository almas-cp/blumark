Capabilities

High-Fidelity Tracking

Powered by multi-strategy scanning and edge AI.

BLE Beacon Protocol

Faculty devices broadcast secure, low-latency Bluetooth beacons. Students scan to instantly verify physical presence within the classroom perimeter.

Real-Time Sync

Instantaneous cloud state synchronization via Supabase Realtime via PostgreSQL. Check-ins are reflected on faculty dashboards with millisecond latency.

AR Head Counting

Integrates Google ML Kit face detection for secondary verification. Perform a 180° camera sweep to algorithmically cross-reference physical attendance.

Role-Based Access Control

Isolated operational contexts for Admins, Faculty, and Students engineered directly into the UI architecture with strict database RLS policies.

Multi-Strategy Scanning

Dynamically fallback across 4 BLE scan modes (Low Latency, Balanced, Low Power, Opportunistic) ensuring deep compatibility across raw Android OEM hardware.

Android Optimized

Built completely in Dart utilizing the Flutter 3.x framework. Compiles directly to native ARM code delivering performance on Android platforms.

Execution Flow

System Mechanics

Faculty Lifecycle

  1. Authenticate securely via Supabase Auth standard protocol.
  2. Configure session metadata (department, batch, structural parameters).
  3. Initiate peripheral BLE broadcast. Device emits tokenized 0000FFF0... UUID.
  4. Monitor real-time webhook responses natively in the dashboard view.
  5. Terminate advertising state to finalize session block into immutable record.
